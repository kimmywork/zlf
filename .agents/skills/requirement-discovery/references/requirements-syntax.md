# Requirements Syntax

Use high-level stories for user value and precise requirement syntax for details.

## User Story

```text
As a <persona>, I want <capability>, so that <benefit>.
```

Use for high-level needs and story maps.

## EARS

Use for precise software/system requirements.

```text
When <trigger>, the system shall <response>.
While <state>, the system shall <response>.
Where <feature is included>, the system shall <response>.
If <condition>, then the system shall <response>.
The system shall <always-valid behavior>.
```

## Given / When / Then

Use for acceptance criteria and tests.

```text
Given <context>,
When <action>,
Then <observable binary result>.
```

## Quality bar

- Binary outcome.
- Observable behavior.
- No vague adjectives without measurement.
- Includes boundary or failure case when risk exists.
