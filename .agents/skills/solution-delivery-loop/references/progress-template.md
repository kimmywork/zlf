# Progress Update Template

Use for phase/version-level progress tracking. Lighter than a delivery record — captures what's done, what's blocked, and decisions made since the last update.

~~~~markdown
# Progress Update: <Feature / Phase / Version>

## Status Overview

- **Phase**: <requirement-discovery / solution-design / implementation-execution / delivery-acceptance>
- **Version / iteration**: <vN>
- **Overall**: <on-track / blocked / needs-review>

## Completed

- <item> — <evidence: test pass / commit hash / manual check>
- <item> — <evidence>

## In Progress

- <item> — <expected completion>

## Blocked

- <item> — <blocker>, <workaround or dependency>

## Decisions Made

- <decision and rationale>
- If any decision changes approved scope, design, acceptance, contract, data model, or verification, also create a change note (`solution-delivery-loop/references/change-note-template.md`).

## Open Questions

- <question>

## Next Action

<single next step>
~~~~
