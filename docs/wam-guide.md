# Prolog Warren 抽象机（WAM）完整实现指南
本指南基于 Hassan Aït-Kaci 的经典著作 *Warren's Abstract Machine: A Tutorial Reconstruction* 整理，遵循**渐进式重构**的思路，从最简的合一机器出发，逐层叠加过程调用、回溯、性能优化等能力，最终还原完整的 WAM 架构。

---

## 1 概述
Warren 抽象机（Warren Abstract Machine, WAM）由 David H. D. Warren 于 1983 年设计，是目前 Prolog 编译器的事实标准执行模型。它定义了一套专门面向逻辑编程的指令集与内存架构，将 Prolog 的 SLD 消解、合一、回溯等核心机制映射为高效的底层操作。

### 1.1 设计核心目标
1.  作为 Prolog 编译器的可移植目标码，可在通用硬件上高效实现
2.  最小化堆内存占用，减少不必要的项复制
3.  通过专用指令与寄存器优化，降低合一与回溯的开销
4.  支持尾调用、环境裁剪等经典编译优化

### 1.2 渐进式实现路线
本指南遵循“从简单到完整”的构建路径，与原教程的分层设计完全对应：
| 阶段 | 机器模型 | 核心能力 |
|------|----------|----------|
| 1 | M₀ 纯合一机器 | 一阶项的堆表示 + 项合一 |
| 2 | M₁ 参数寄存器机器 | 谓词过程调用 + 事实匹配 |
| 3 | M₂ 平面消解机器 | 规则体执行 + 环境栈 + 永久变量 |
| 4 | M₃ 纯 Prolog 机器 | 多子句回溯 + 选择点 + 追踪栈 |
| 5 | 完整 WAM | 全量优化：专用指令、尾调用、索引、Cut 等 |

## 2 核心基础：一阶项与合一
WAM 的本质是一台高效的合一机器。所有 Prolog 执行最终都可规约为项的合一与回溯。

### 2.1 一阶项定义
一阶项（term）分为三类：
- **变量**：大写字母开头，如 `X`, `Y`，运行时可绑定到其他项
- **常量**：小写字母开头的原子或数字，如 `a`, `123`，是 0 元函子
- **结构**：形式为 `f(t₁, t₂, ..., tₙ)`，其中 `f` 为函子（functor），`n` 为元数（arity），`tᵢ` 为子项

### 2.2 最广合一子（MGU）
合一（unification）是求解项等式的过程：找到一个变量替换 σ，使得两个项应用 σ 后完全相同，其中最通用的替换称为**最广合一子**（Most General Unifier, MGU）。

WAM 采用基于 Union-Find 思想的迭代合一算法，通过**解引用（deref）** 追踪变量绑定链，通过**追踪栈（trail）** 支持绑定撤销。

## 3 阶段一：纯合一机器 M₀
M₀ 是 WAM 的最小内核，仅负责项的内存表示与合一执行。

### 3.1 项的堆表示
所有项存储在全局可寻址堆（HEAP）中，堆是一个连续数组，每个堆单元格包含标签与数据两部分。初始定义两类标签：
1.  **REF（引用）**：表示变量，数据为指向另一个堆单元格的地址；未绑定变量为**自引用**（地址指向自身）
2.  **STR（结构）**：表示结构指针，数据指向函数字的地址；函数字存储 `f/n`（函子名+元数），后续紧跟 n 个连续的子项指针

> 示例：项 `p(Z, h(Z,W), f(W))` 的堆布局
> - 结构 `p/3` 的 STR 指针指向函数字 `p/3`，后续 3 个槽分别指向 Z、h/2、f/1
> - 变量 Z、W 各对应一个 REF 单元格，多次出现共享同一个地址

### 3.2 核心寄存器
- `H`：堆顶指针，始终指向堆中下一个可用单元格
- `S`：结构扫描指针，读模式下指向当前待匹配的子项
- `X₁, X₂, ...`：临时寄存器，暂存项的堆地址，用于项构建过程中的变量共享

### 3.3 查询编译：构建项
查询（goal）的编译目标是生成指令序列，在堆上构建出对应的项结构。核心指令共 3 条：
1.  `put_structure f/n, Xi`：在堆上压入函数字 `f/n`，将 STR 指针存入寄存器 `Xi`
2.  `set_variable Xi`：压入一个新的未绑定 REF 单元格，存入寄存器 `Xi`
3.  `set_value Xi`：压入一个单元格，内容为寄存器 `Xi` 的值（用于变量多次出现）

### 3.4 程序编译：匹配项
程序（子句头）的编译目标是与堆上已有的查询项进行合一，运行在**双模式**下：
- **读模式**：查询项已实例化，按结构逐一匹配
- **写模式**：查询项是未绑定变量，直接在堆上构建程序项

核心指令共 3 条：
1.  `get_structure f/n, Xi`：解引用 `Xi`，若为未绑定变量则绑定到新结构并切换写模式；若为同函子结构则切换读模式并设置 `S` 指针；否则合一失败
2.  `unify_variable Xi`：读模式下将 `S` 指向的堆内容存入 `Xi`；写模式下压入新变量并存入 `Xi`；最后 `S` 自增
3.  `unify_value Xi`：读模式下将 `Xi` 与 `S` 指向的项合一；写模式下压入 `Xi` 的值；最后 `S` 自增

### 3.5 核心辅助操作
#### （1）解引用 deref
沿着 REF 指针链一直追溯，直到遇到非 REF 单元格或未绑定的自引用 REF 单元格。这是所有项访问的前置操作。
```
function deref(a):
    if STORE[a] is REF with value v:
        return deref(v)
    else:
        return a
```

#### （2）绑定 bind
将一个未绑定变量绑定到另一个项，直接修改变量 REF 单元格的地址值。回溯时需要撤销的绑定会先记录到追踪栈。

#### （3）合一 unify
基于下推栈（PDL, Push-Down List）的迭代合一算法，避免递归开销：
1.  将两个项的初始地址压入 PDL
2.  循环弹出一对地址，解引用后比较
3.  若有变量则执行绑定；若为同结构则将所有子项地址依次压入 PDL
4.  若结构不匹配则合一失败

## 4 阶段二：平面消解机器 M₁ / M₂
在 M₀ 基础上引入过程调用、规则体与环境栈，实现无回溯的 Prolog 子集。

### 4.1 M₁：参数寄存器与过程调用
#### 参数寄存器约定
谓词的 n 个参数通过前 n 个临时寄存器传递，称为**参数寄存器**，记为 `A₁ ... Aₙ`（与 `X₁ ... Xₙ` 物理复用，仅语义区分）。

新增 4 条参数操作指令：
- 查询端（传参）：`put_variable Xi Ai`、`put_value Xi Ai`
- 程序端（接参）：`get_variable Xi Ai`、`get_value Xi Ai`

#### 控制流指令
- `P` 寄存器：程序计数器，指向下一条待执行指令
- `CP` 寄存器：续行指针，保存过程调用后的返回地址
- `call p/n`：保存当前 `P` 到 `CP`，跳转到谓词 `p/n` 的代码入口
- `proceed`：过程结束，跳转到 `CP` 指向的返回地址

### 4.2 M₂：规则、永久变量与环境栈
#### 永久变量 vs 临时变量
- **临时变量**：仅出现在单个目标中，调用后即销毁，可存在寄存器里
- **永久变量**：出现在多个体目标中，会被后续过程调用覆盖，必须保存到栈上

#### 环境栈与环境帧
环境栈（STACK）用于存储过程激活帧（环境帧），每个规则激活对应一个环境帧。环境帧结构：
| 偏移 | 字段 | 含义 |
|------|------|------|
| 0 | CE | 前一环境指针（Continuation Environment） |
| 1 | CP | 续行地址 |
| 2+ | Y₁ ... Yₙ | 永久变量槽位 |

新增指令：
- `allocate N`：在栈上分配包含 N 个永久变量的环境帧，设置 CE、CP 字段，更新环境指针 `E`
- `deallocate`：弹出当前环境帧，恢复 `E` 和 `CP`

#### 规则编译模板
```
allocate N        ; 分配环境帧
<头部 get 指令>    ; 合一子句头参数
<目标1 put 指令>
call p1/n1
...
<目标k put 指令>
call pk/nk
deallocate        ; 释放环境帧
proceed
```

## 5 阶段三：完整纯 Prolog 机器 M₃
引入选择点、追踪栈与回溯机制，实现完整的 SLD 深度优先消解。

### 5.1 选择点与单栈设计
当谓词包含多个子句时，调用时需要创建**选择点（choice point）**，保存当前计算状态，以便失败时回溯尝试下一个子句。

WAM 采用**单栈设计**：环境帧与选择点帧共用同一个栈，选择点会“盖住”其下方的环境帧，防止环境提前释放后被覆盖，保证回溯时能正确恢复状态。

选择点帧结构：
| 偏移 | 字段 | 含义 |
|------|------|------|
| 0 | arity | 谓词元数 |
| 1 ~ n | A₁ ... Aₙ | 参数寄存器快照 |
| n+1 | CE | 当前环境指针 E |
| n+2 | CP | 当前续行指针 CP |
| n+3 | B | 前一选择点指针 |
| n+4 | BP | 下一个待尝试子句的地址 |
| n+5 | TR | 追踪栈顶快照 |
| n+6 | H | 堆顶快照 |
| n+7 | B0 | Cut 指针快照 |

核心寄存器：
- `B`：当前最新选择点的指针
- `HB`：最新选择点创建时的堆顶地址，用于判断绑定是否需要追踪

### 5.2 追踪栈（TRAIL）
回溯时需要撤销失败路径上的所有变量绑定，但并非所有绑定都需要记录：
- **无条件绑定**：选择点创建后新产生的变量绑定，回溯时直接随堆指针回退销毁，无需记录
- **条件绑定**：选择点创建前就存在的变量绑定，必须记录到追踪栈，回溯时手动重置为未绑定

`trail(a)` 操作：若地址 `a` 小于 `HB`（堆变量）或小于 `B`（栈变量），则将 `a` 压入追踪栈。

### 5.3 选择点操作指令
针对多子句谓词的开头、中间、末尾子句，对应三条指令：
1.  `try_me_else L`：第一个子句执行前调用，创建完整选择点，设置下一个子句地址为 L，更新 B
2.  `retry_me_else L`：回溯到当前选择点，恢复寄存器、堆、追踪栈状态，更新下一个子句地址为 L
3.  `trust_me`：最后一个子句执行前调用，恢复状态后销毁当前选择点，B 指向前一选择点

### 5.4 回溯操作 backtrack
执行失败时触发：
1.  若没有可用选择点（B 为空），则整体查询失败
2.  否则恢复当前选择点保存的状态，撤销追踪栈中的绑定，回退堆指针
3.  跳转到下一个待尝试子句的代码入口

## 6 阶段四：WAM 全量优化
从朴素的 M₃ 出发，通过一系列空间与时间优化，得到最终的完整 WAM 架构。所有优化都遵循三条核心设计原则：
1.  尽可能节省堆空间
2.  优化寄存器分配，减少不必要的数据移动
3.  高频场景提供专用指令，用特例换性能

### 6.1 堆表示优化
去掉每个结构前冗余的 STR 单元格，改为由指针本身携带 STR 标签，直接指向函数字。仅需修改 `put_structure` 等少数指令，即可节省一个堆单元格，同时减少一次内存间接访问。

### 6.2 专用数据类型指令
为高频出现的常量、列表、匿名变量提供专用标签与指令，避免通用结构指令的开销。
1.  **常量**：新增 `CON` 标签，对应 `put_constant`、`get_constant`、`set_constant`、`unify_constant`，无需为 0 元函子分配结构
2.  **列表**：新增 `LIS` 标签，对应 `put_list`、`get_list`，专门优化 `./2` 列表构造器
3.  **匿名变量**：`set_void n`、`unify_void n`，批量创建/跳过 n 个未命名变量，无需分配寄存器

### 6.3 寄存器分配优化
消除冗余的数据移动：
- 若变量恰好分配在参数寄存器上，省略 `get_variable Xi Ai` 这类自赋值指令
- 尽量复用参数寄存器作为临时变量，减少寄存器间搬移
- 通过窥孔优化消除连续的无用加载/存储

### 6.4 尾调用优化（Last Call Optimization, LCO）
#### 原理
规则体的最后一个目标调用前，当前环境的所有永久变量都已使用完毕，可以提前释放环境帧，再执行最后一次调用。

#### 实现
- 最后一个目标不用 `call`，改用 `execute p/n`：不设置新的 CP，直接跳转
- 将 `deallocate` 提前到 `execute` 之前执行
- 尾递归场景下，栈空间不会增长，程序等价于迭代执行

#### 链规则特例
只有一个体目标的规则（链规则），所有变量都是临时变量，完全不需要环境帧，直接编译为：
```
<头部 get 指令>
<体目标 put 指令>
execute p/n
```

### 6.5 环境裁剪（Environment Trimming）
#### 原理
永久变量并非全程都需要：每个变量都有其最后一次出现的目标，之后即可释放。环境裁剪让环境帧的大小随执行逐步缩小，最大化栈空间复用。

#### 实现
- 按“最后一次出现的目标序号”对永久变量排序，越晚释放的变量在帧中位置越靠下
- 每个 `call` 指令携带参数 N，表示调用后仍需保留的永久变量数量
- 后续栈分配时，可复用已裁剪掉的槽位

### 6.6 栈变量与不安全变量
#### 栈变量
永久变量可以直接存储在栈环境帧中，不必分配堆单元格，大幅节省堆空间。`put_variable Yi Ai` 直接让参数寄存器指向栈上的变量槽。

#### 不安全变量问题
栈变量会随环境裁剪/LCO被释放，如果此时变量仍未绑定且被外部引用，会产生**悬挂指针**。这类变量称为**不安全变量**。

#### 解决方案：绑定方向规则
WAM 通过三条绑定规则从根本上避免堆指向栈：
1.  高地址变量永远指向低地址变量
2.  堆变量绝不允许指向栈变量
3.  内存布局上，栈的地址整体高于堆

#### 运行时安全检查
- `put_unsafe_value Yi Ai`：最后一次使用不安全变量前检查，若仍为未绑定的栈变量，则复制到堆上再传递
- `set_local_value Xi` / `unify_local_value Xi`：结构内部引用栈变量时的安全处理，必要时全局化到堆

### 6.7 子句索引（Indexing）
#### 原理
根据谓词第一个参数的类型与值，直接跳转到可能匹配的子句，避免为所有子句创建选择点，大幅减少回溯开销。

#### 三级索引架构
1.  **一级分派**：`switch_on_term V, C, L, S`
    根据第一个参数解引用后的类型（变量/常量/列表/结构），跳转到对应分支
2.  **二级分派**：`switch_on_constant` / `switch_on_structure`
    通过哈希表按具体常量值或函子名，跳转到对应子句组
3.  **三级串行**：`try` / `retry` / `trust`
    同一键值对应多个子句时，串行尝试，逻辑同 `try_me_else` 系列

### 6.8 Cut（截断）的实现
Cut 是 Prolog 的控制原语，作用是丢弃当前谓词进入后创建的所有选择点，提交已做的绑定。

#### 核心机制
- 新增 `B0` 寄存器：调用谓词时，将当前选择点 B 保存到 B0
- Cut 执行时，将 B 重置为 B0，丢弃中间所有选择点，并清理追踪栈中对应的无条件绑定

#### 两类 Cut
1.  **浅 Cut（颈部 Cut）**：紧跟在子句头部之后，用 `neck_cut` 指令直接操作 B0
2.  **深 Cut**：出现在体目标中间，需要用 `get_level Yn` 将 B0 保存到环境中，再用 `cut Yn` 执行截断

## 7 完整 WAM 架构总览
### 7.1 全局内存布局（低地址 → 高地址）
1.  **代码区（CODE）**：存储编译后的 WAM 指令序列
2.  **堆（HEAP）**：存储项结构，向上增长
3.  **栈（STACK）**：存储环境帧与选择点帧，向下增长（整体地址高于堆）
4.  **追踪栈（TRAIL）**：记录条件绑定地址
5.  **PDL 栈**：合一算法的临时工作栈

### 7.2 核心寄存器汇总
| 类别 | 寄存器 | 功能 |
|------|--------|------|
| 程序控制 | P | 程序计数器 |
| | CP | 续行返回地址 |
| | B0 | Cut 基准选择点指针 |
| 栈管理 | E | 当前环境帧指针 |
| | B | 当前选择点指针 |
| 堆管理 | H | 堆顶指针 |
| | HB | 最新选择点的堆顶快照 |
| | S | 结构扫描指针 |
| 参数/临时 | A₁...Aₙ / X₁...Xₙ | 参数传递与临时变量 |
| 回溯 | TR | 追踪栈顶指针 |
| 模式 | mode | 合一的读/写模式 |

### 7.3 完整指令集分类
| 类别 | 主要指令 | 作用 |
|------|----------|------|
| Put 类 | put_variable, put_value, put_structure, put_list, put_constant, put_unsafe_value | 调用端构建项、传递参数 |
| Get 类 | get_variable, get_value, get_structure, get_list, get_constant | 子句头匹配项、接收参数 |
| Set 类 | set_variable, set_value, set_local_value, set_constant, set_void | 结构内部写操作 |
| Unify 类 | unify_variable, unify_value, unify_local_value, unify_constant, unify_void | 结构内部读写双模式操作 |
| 控制类 | allocate, deallocate, call, execute, proceed | 过程调用与环境管理 |
| 选择点类 | try_me_else, retry_me_else, trust_me, try, retry, trust | 选择点创建与回溯 |
| 索引类 | switch_on_term, switch_on_constant, switch_on_structure | 子句快速分派 |
| Cut 类 | neck_cut, get_level, cut | 截断操作 |

## 8 从零实现 WAM 的步骤建议
1.  **实现 M₀ 纯合一机**：先完成堆内存、REF/STR 标签、deref、bind、基础合一算法，以及 put/get/set/unify 基础指令，验证两个项的合一结果。
2.  **实现 M₁ 过程调用**：加入参数寄存器、P/CP 寄存器、call/proceed 指令，实现事实的查询与匹配。
3.  **实现 M₂ 规则与环境**：实现环境栈、allocate/deallocate，完成永久/临时变量划分，支持多目标规则执行。
4.  **实现 M₃ 回溯机制**：实现选择点帧、try/retry/trust 指令、追踪栈、unwind_trail 与 backtrack，支持多子句回溯。
5.  **逐项加入优化**：按“堆表示优化 → 专用数据指令 → 寄存器优化 → 尾调用 → 环境裁剪 → 栈变量与安全检查 → 索引 → Cut”的顺序逐步实现，每一步都做充分测试。
6.  **完善周边能力**：实现 Prolog 语法解析、内置谓词、项输出，可选加入垃圾回收。

## 9 常见实现坑点与注意事项
1.  **解引用不可遗漏**：任何访问项内容的操作前，都必须先对地址执行 deref，否则会读到中间 REF 指针导致错误。
2.  **绑定方向必须严格**：违反“堆不指向栈”的规则会导致环境释放后出现悬挂指针，引发内存错误。
3.  **不安全变量不能遗漏检查**：环境裁剪和尾调用场景下，首次出现在体目标中的永久变量必须做安全处理。
4.  **选择点的环境保护**：deallocate 仅逻辑上释放环境，若栈顶是选择点，栈空间不能实际回收。
5.  **Trail 条件判断**：只有选择点创建前就存在的变量，其绑定才需要追踪，避免冗余记录。
6.  **索引后的冗余检查**：switch 分派后对应的 get 指令会重复检查类型，可通过优化消除冗余。
7.  **回溯时恢复 B0**：选择点帧必须保存 B0，否则回溯后 Cut 会使用错误的基准指针。
