# Change Note: Persistent tabling is mandatory

## Trigger

The first 10K stress run demonstrated that the current positive fixed-point prototype and shared in-memory table are insufficient evidence for scale. User review reaffirmed the previously discussed memory + RocksDB two-level requirement.

## Decision

Before 100K, 1M, or full-data stress runs, tabling must provide:

- hot in-memory table lookup;
- complete-table RocksDB persistence and restart reload;
- atomic answer/metadata publication;
- interrupted evaluating tables treated as stale;
- nested tabled subgoals, recursive SCC completion, and delta scheduling;
- indexed call-time provider dispatch and metrics.

Live WAM heap/register/continuation frames remain process-local and are never persisted.

## Impact

The existing 10K report is retained as an exploratory baseline, not acceptance evidence. Larger stress tiers are paused until the tabling acceptance tests pass.
